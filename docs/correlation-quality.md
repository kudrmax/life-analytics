# Качество корреляционных пар

Каждая пара в отчёте корреляций может иметь причину ненадёжности (`quality_issue`).
NULL — пара статистически надёжна. NOT NULL — есть проблема.

## Причины

| Значение | Условие | Что означает |
|---|---|---|
| `low_data_points` | < 10 общих дней данных | Мало наблюдений |
| `insufficient_variance` | variance=0 (константа, любой тип) или бинарный с variance ≤ 0.10 | Значение почти не меняется, корреляция случайна |
| `high_p_value` | p-value ≥ 0.05 | Статистически незначимо |

## Приоритет

Подходить может несколько причин — записывается одна, наиболее приоритетная.
Порядок задаётся списком `checks` в `_determine_quality_issue()` — первое совпадение побеждает.

1. `low_data_points` — при малом n всё остальное бессмысленно
2. `insufficient_variance` — при низкой дисперсии даже p < 0.05 ненадёжен
3. `high_p_value` — данных и вариации достаточно, но связь не подтверждена

## Детали insufficient_variance

Для каждого источника вычисляется `variance(vals)`. Два случая:
- **variance = 0** (любой тип): источник константен (stdev=0). Без этой проверки `_compute_pearson` возвращал r=0.0 и пара попадала в "слабые" вместо "незначимых".
- **Бинарный + variance ≤ 0.10**: бинарный источник (все значения 0.0/1.0) с перекосом ~88%+ дней одинаковые. Для бинарных variance ≈ p×(1−p). Несколько "редких" дней случайно совпадают с другой метрикой → ложный r.

Непрерывные источники с variance > 0 не затрагиваются — при низкой дисперсии p-value уже высокий.
