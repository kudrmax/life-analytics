# Life Analytics

Персональный трекер ежедневных метрик. Гибкая настройка метрик, многоразовые записи в течение дня, тренды и корреляции.

## Возможности

- **21 метрика из коробки** — сон, настроение, энергия, тренировки, питание, продуктивность, финансы, социальное
- **Многоразовые метрики** — настроение, энергия и стресс можно отмечать несколько раз в день с автоматической агрегацией (среднее, мин/макс)
- **Составные метрики** — например, "Алкоголь: да/нет + количество порций" с условными полями
- **Гибкая настройка** — добавление/удаление/отключение метрик через интерфейс
- **Аналитика** — тренды за период, корреляции между метриками (Пирсон), стрики
- **История** — календарь с просмотром сводки за любой день

## Запуск

```bash
# 1. Установка зависимостей (один раз)
cd backend
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt

# 2. Запуск
cd ..
./run.sh
```

- Frontend: http://localhost:3000
- API: http://localhost:8000
- API docs: http://localhost:8000/docs

## Настройка дефолтных метрик

При регистрации каждый новый пользователь получает 22 метрики по умолчанию (сон, настроение, тренировки, питание и т.д.).

### Как изменить дефолтные метрики:

1. Откройте файл `backend/app/seed.py`
2. Найдите список `DEFAULT_METRICS = [`
3. Добавьте, измените или удалите метрики
4. Перезапустите backend

**Пример добавления метрики:**
```python
{
    "id": "water",
    "name": "Стаканов воды",
    "category": "Здоровье",
    "type": "number",
    "frequency": "daily",
    "config": {"min": 0, "max": 20, "label": "стаканов"},
}
```

**Типы метрик:**
- `scale` — шкала 1-5
- `boolean` — да/нет
- `number` — число
- `time` — время (HH:MM)
- `enum` — список опций
- `compound` — составная метрика с условными полями

**Частота записи:**
- `daily` — одна запись в день
- `multiple` — три записи в день (утро/день/вечер)

⚠️ **Важно:** Изменения применяются только для новых пользователей. Существующие пользователи могут добавить метрики через интерфейс (Настройки).

## Экспорт и импорт данных

Все данные (метрики + записи) можно экспортировать и импортировать через интерфейс:
- **Экспорт**: Настройки → "📥 Экспорт ZIP" — скачивается архив с metrics.csv и entries.csv
- **Импорт**: Настройки → "📤 Импорт ZIP" — загрузка данных из архива

Это полезно для:
- Резервного копирования
- Переноса данных между аккаунтами
- Анализа в Excel/Google Sheets

## Автоматические бэкапы базы данных

Life Analytics может автоматически сохранять резервные копии базы данных на Яндекс Диск. Бэкапы создаются каждые 6 часов и хранятся 30 дней.

### Как это работает

- Отдельный контейнер делает дамп PostgreSQL (`pg_dump`), сжимает его и загружает на Яндекс Диск
- Старые бэкапы автоматически удаляются после истечения срока хранения
- При локальной разработке бэкапы **не запускаются** — только на сервере

### Настройка

**1. Получите OAuth-токен Яндекс Диска:**

Перейдите на https://yandex.ru/dev/disk/poligon/ и получите токен с правами на запись.

**2. Добавьте токен в `.env`:**

```
YADISK_TOKEN=ваш-токен
```

**3. Запустите сервис бэкапов:**

```bash
make backup-up
```

Первый бэкап создастся сразу при запуске, далее — каждые 6 часов.

### Управление

```bash
make backup-up       # запустить сервис бэкапов
make backup-down     # остановить
make backup-logs     # посмотреть логи
make backup-now      # сделать бэкап прямо сейчас (разово)
```

### Настраиваемые параметры

Все параметры задаются через переменные окружения в `.env`:

| Параметр | По умолчанию | Описание |
|---|---|---|
| `YADISK_TOKEN` | — | OAuth-токен Яндекс Диска (обязательный) |
| `YADISK_BACKUP_PATH` | `/life-analytics-backups/` | Папка на Яндекс Диске |
| `BACKUP_INTERVAL_MINUTES` | `360` (6 часов) | Интервал между бэкапами |
| `BACKUP_RETAIN_DAYS` | `30` | Сколько дней хранить бэкапы |

### Восстановление из бэкапа

1. Скачайте нужный файл с Яндекс Диска (формат `life_analytics_YYYY-MM-DD_HH-MM-SS.sql.gz`)
2. Распакуйте и загрузите в БД:

```bash
gunzip life_analytics_2026-03-04_12-00-00.sql.gz
docker exec -i life-analytics-db-1 psql -U la_user -d life_analytics < life_analytics_2026-03-04_12-00-00.sql
```
