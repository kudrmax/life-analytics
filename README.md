# Life Analytics

Персональный трекер ежедневных метрик с трендами, корреляциями и интеграциями.

---

## Для пользователей

### Что это

Записывайте каждый день: сон, настроение, тренировки, питание — любые показатели, которые важны вам. Смотрите тренды, находите закономерности, делайте выводы. Мультипользовательский — каждый заводит свой аккаунт.

### Возможности

- **Гибкие метрики** — создавайте свои с нужным типом ввода
- **Несколько замеров в день** — утро, день, вечер (слоты)
- **Категории** — группируйте метрики (Здоровье, Продуктивность, ...)
- **Условные метрики** — показывать «Количество порций» только если «Алкоголь = да»
- **Формулы** — автоматически считайте одни метрики на основе других
- **Тренды** — графики и статистика за любой период
- **Корреляции** — автоматический поиск связей между метриками (что влияет на что)
- **Выводы** — записывайте свои наблюдения и привязывайте к метрикам
- **Заметки** — свободный текст для метрик, где число не подходит
- **История** — календарь с просмотром сводки за любой день
- **Экспорт/импорт** — все данные в ZIP (CSV), анализ в Excel

### Типы метрик

| Тип | Пример |
|---|---|
| Да/Нет | Тренировка была? |
| Число | Стаканов воды |
| Шкала | Настроение 1–10 |
| Время | Во сколько лёг спать |
| Длительность | Сколько тренировался |
| Выбор из списка | Вид тренировки (бег, зал, йога) |
| Формула | Среднее настроение = (утро + вечер) / 2 |
| Текст/Заметки | Свободная запись |

### Интеграции

- **Todoist** — автоматический подсчёт выполненных задач, фильтры и запросы
- **ActivityWatch** — экранное время с компьютера (приложения, сайты, категории)

### Экспорт и импорт

Все данные можно выгрузить в ZIP (CSV-файлы) и загрузить обратно. Подходит для бэкапов, переноса между аккаунтами и анализа в Excel.

### Конфиденциальность

Режим приватности скрывает чувствительные метрики — маскирует название и значение, чтобы можно было показать экран кому-то.

## Для разработчиков

### Быстрый старт (Docker Compose)

```bash
cp .env.example .env
make up
```

- Frontend: http://localhost:3000
- Backend: http://localhost:8000
- API docs: http://localhost:8000/docs

### Локальный запуск (без Docker)

Нужен локальный PostgreSQL с базой `life_analytics`.

```bash
# Backend
cd backend
python3 -m venv venv && source venv/bin/activate
pip install -r requirements.txt
python -m uvicorn app.main:app --reload --port 8000

# Frontend (в отдельном терминале)
cd frontend
python -m http.server 3000
```

### Автоматические бэкапы

Отдельный контейнер делает дамп PostgreSQL (`pg_dump`), сжимает и загружает на Яндекс Диск. Старые бэкапы удаляются автоматически.

**Настройка:**

1. Получите OAuth-токен на https://yandex.ru/dev/disk/poligon/
2. Добавьте в `.env`: `YADISK_TOKEN=ваш-токен`
3. `make backup-up` — первый бэкап создастся сразу, далее каждые 6 часов

| Параметр | По умолчанию | Описание |
|---|---|---|
| `YADISK_TOKEN` | — | OAuth-токен Яндекс Диска (обязательный) |
| `YADISK_BACKUP_PATH` | `/life-analytics-backups/` | Папка на Яндекс Диске |
| `BACKUP_INTERVAL_MINUTES` | `360` (6 часов) | Интервал между бэкапами |
| `BACKUP_RETAIN_DAYS` | `30` | Сколько дней хранить бэкапы |

**Восстановление:**

```bash
make backup-restore FILE=backups/life_analytics_2026-03-04_12-00-00.sql.gz
```

### Деплой

Docker Compose на VPS. CI/CD через GitHub Actions — push в master автоматически деплоит на сервер.

### Стек

- **Backend:** Python, FastAPI, asyncpg, PostgreSQL 16
- **Frontend:** Vanilla JS SPA
- **Деплой:** Docker Compose, nginx, GitHub Actions CI/CD
- **Аутентификация:** JWT (bcrypt + 7-дневные токены)
