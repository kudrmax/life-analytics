# Концептуальные улучшения системы Life Analytics

## 1. Эксперименты (A/B-тестирование на себе)

Сейчас система **наблюдает**: собирает данные и ищет корреляции. Но корреляция ≠ причинность. Нужен механизм **интервенций**.

**Концепция:** Пользователь создаёт эксперимент: "2 недели пью 8 стаканов воды → смотрю влияние на энергию и фокус". Система автоматически:
- Делит период на baseline (до) и intervention (после)
- Считает статистику по целевым метрикам в обоих периодах
- Показывает значимость разницы (t-test / Mann-Whitney)
- Хранит историю экспериментов с выводами

**Почему это трансформационно:** Превращает пассивный трекер в инструмент принятия решений. Вместо "кофе коррелирует с плохим сном" → "я проверил: без кофе после 14:00 сон улучшился на 1.2 балла, p=0.03".

---

## 2. Временные паттерны за пределами lag=1

Текущий движок видит только "вчера → сегодня". Это малая часть реальности.

**Что теряется:**
- **Кумулятивные эффекты:** 3 дня недосыпа подряд → обвал на 4-й (lag=1 этого не поймает)
- **Недельные циклы:** Продуктивность системно падает к пятнице
- **Скользящие окна:** Среднее настроение за последние 3 дня — лучший предиктор завтрашней энергии, чем вчерашнее значение
- **Пороговые эффекты:** Кофе ≤ 2 шота — нет влияния, ≥ 3 — резкое падение сна (линейная корреляция этого не поймает)

**Концепция:** Расширить аналитический движок:
- Скользящие средние (3/7/14 дней) как авто-источники
- Streak-источники ("N дней подряд без X")
- Нелинейная корреляция: разбивка на квантили + сравнение групп вместо Pearson r

## 3. Аномалии и алерты

Система молчит, пока ты сам не зайдёшь и не посмотришь. А должна подсказывать.

**Концепция:** На странице "Сегодня" — блок "Что интересного":
- **Аномалии:** "Энергия утром 2/5 — на 1.5σ ниже твоей нормы за последний месяц"
- **Серии:** "Это 5-й день без тренировки, среднее — раз в 2 дня"
- **Нарушенные паттерны:** "Обычно по вторникам mood=4, сегодня 2"
- **Предупреждения из корреляций:** "Вчера кофе после 16:00 — исторически это -1.3 к качеству сна"

Не ML, а простая статистика: z-scores, контрольные карты, персональные базлайны.

---

## 4. LLM-анализ данных (AI-инсайты)

Структурированные данные + текстовые заметки. LLM может находить паттерны, которые числовой движок не видит.

**Концепция:** Периодический (раз в неделю) или по запросу запуск анализа:
- На вход: числовые тренды + текстовые заметки + корреляционный отчёт
- LLM формулирует наблюдения на естественном языке: "На прошлой неделе три дня с высоким стрессом совпали с записями про дедлайн проекта X. При этом тренировка в эти дни снижала стресс к вечеру."
- Результат сохраняется как insight с привязкой к метрикам

**Почему ценно:** Текстовые метрики (переживания, благодарность, планы) сейчас участвуют в корреляциях только через `note_count` — содержание игнорируется. LLM может извлечь семантику.

## 5. Композитные индексы с обратной связью

Computed-метрики сейчас — это статичные формулы. Но "идеальная формула благополучия" у каждого своя.

**Концепция:** Система предлагает оптимальные веса для составного индекса на основе данных:
- Пользователь выбирает целевую метрику (например, "продуктивность")
- Система перебирает входные метрики и находит те, которые объясняют максимум дисперсии (простая линейная регрессия)
- Предлагает формулу: "Продуктивность ≈ 0.4×сон + 0.3×зарядка + 0.2×energy_утро − 0.1×стресс"
- Пользователь может принять и создать computed-метрику

---

## 6. Контекстные режимы / фазы жизни

Сейчас все дни "одинаковые" для аналитики. Но жизнь имеет фазы: отпуск, болезнь, интенсивный рабочий период, выходные.

**Концепция:** Теги дней или режимы:
- Пользователь отмечает: "отпуск", "болезнь", "командировка"
- Корреляции считаются **раздельно** по контекстам
- "Кофе → сон" в рабочие дни vs в отпуске — разные паттерны
- `fill_rate` и аномалии учитывают контекст (низкий fill_rate в отпуске — норма)

Это решает фундаментальную проблему: усреднение по разным режимам жизни размывает реальные связи.

---

## 7. Предиктивный слой

Когда данных достаточно (3+ месяца), можно перейти от "что было" к "что будет".

**Концепция:** Лёгкая прогнозная модель:
- "Исходя из того, что ты поспал 5 часов, не делал зарядку и выпил 4 кофе — вероятность низкой энергии вечером 78%"
- Не deep learning — простая логистическая регрессия или решающее дерево на персональных данных
- Помогает принимать решения в моменте: "Стоит ли пить 4-й кофе?"

---

## 8. Граф причинности

Сейчас корреляции — плоский список пар. Но метрики образуют **граф**: сон → энергия → продуктивность → удовлетворённость. Или: стресс → плохой сон → низкая энергия → прокрастинация → больше стресса (петля).

**Концепция:** Визуализация в виде направленного графа:
- Узлы — метрики, рёбра — значимые корреляции (с lag → направление)
- Выделение циклов (feedback loops) — самые ценные для интервенций
- Кликабельный: нажал на ребро → детали корреляции
- Помогает найти "рычаги" — метрики, которые влияют на многие другие

---

## Приоритеты (ценность / сложность)

1. **Аномалии и алерты** (#3) — минимум кода, максимум ежедневной пользы. z-scores + простые правила
2. **Новые авто-источники** (slot_delta/range/fill_rate) — бесплатные данные, ~50 строк
3. **Контекстные режимы** (#6) — одна таблица + фильтр в корреляциях, принципиально меняет качество анализа
4. **Эксперименты** (#1) — средняя сложность, превращает трекер в decision-making tool
5. **LLM-анализ** (#4) — с claude-agent-sdk относительно просто, выхлоп от текстовых данных огромный
6. **Временные паттерны** (#2) — расширение движка, средняя сложность
7. **Композитные индексы** (#5) — требует UI для выбора целевой метрики и визуализации весов
8. **Граф причинности** (#8) — мощно, но требует визуализации (D3/force-graph)
9. **Предиктивный слой** (#7) — нужно 3+ месяца данных, scipy/sklearn на бэкенде
